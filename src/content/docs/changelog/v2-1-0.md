# AgentSecrets v2.1.0

## What's New

### `agentsecrets docs` Command
Search and read the official AgentSecrets documentation directly from your terminal — no browser required.

```bash
# Search for a specific topic
agentsecrets docs proxy

# Use natural language queries
agentsecrets docs how to share credentials with my team securely
```

Results are presented in an interactive selector. Choose an article and it renders the precise content inline.

---

## Fixes & Improvements

### Keychain-Auth Protocol Mismatch Auto-Recovery
When upgrading from v1.x to v2.x, the `keychain-auth` daemon could be left running the old v1 protocol. Previously this surfaced as `unexpected response status ""` in `agentsecrets status`. The CLI now detects protocol mismatches during initialization and automatically restarts the daemon to the correct version — no manual intervention needed.

### Telemetry Sync: Empty UUID Fix
Telemetry snapshots collected before a workspace or project was selected could include empty strings for `workspace_id` and `project_id`, causing the backend to reject them with a `400: Must be a valid UUID` error. These fields are now omitted when empty.

### `secrets pull`: False Conflict Warning
Running `agentsecrets secrets pull` when secrets existed in the cloud but not locally incorrectly displayed a "Local changes detected that will be overwritten" warning. The pull now correctly identifies these as new additions, not conflicts.
